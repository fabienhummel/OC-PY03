# Analyse de la note de lancement HomeSkolar

## Objectif

Extraire uniquement les besoins explicitement formulés dans la note de lancement, sans ajouter d'hypothèse ni de solution technique.

## Contexte métier

HomeSkolar met en relation des enfants en difficulté scolaire avec des tuteurs bénévoles à distance. Chaque élève inscrit se voit assigner un tuteur qui l'accompagne lors de courts rendez-vous hebdomadaires afin de l'aider à faire ses devoirs et à mieux s'organiser.

## Acteurs confirmés

- élève ;
- tuteur bénévole.

## Domaines fonctionnels confirmés

1. Gestion du compte utilisateur
2. Communication
3. Rencontres et calendrier
4. Gestion des tâches

## Sous-fonctionnalités extraites

### Gestion du compte

- inscription ;
- connexion ;
- gestion du mot de passe ;
- gestion des données personnelles.

### Communication

- échange et consultation de messages ;
- possibilité pour l'élève d'épingler certains messages ;
- notification des messages non lus.

### Rencontres et calendrier

- planification des rencontres ;
- affichage des rendez-vous et événements ;
- consultation dans une page de calendrier classique.

### Gestion des tâches

- consultation des tâches à réaliser avant la rencontre suivante ;
- notification de ces tâches dans l'application ;
- création d'une tâche personnelle par l'élève ou le tuteur ;
- usage possible comme note ou mémo.

## Éléments non déduits

La note ne précise pas notamment :

- l'existence d'un administrateur ;
- les modalités détaillées d'affectation des tuteurs ;
- les pièces jointes ou la visioconférence ;
- les règles détaillées de modification ou d'annulation des rendez-vous ;
- les priorités, échéances ou statuts détaillés des tâches ;
- les technologies à utiliser.

Ces éléments ne sont pas ajoutés aux spécifications fonctionnelles.
