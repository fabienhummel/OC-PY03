# Spécifications fonctionnelles

Cette section regroupe les documents de travail et les livrables intermédiaires produits pendant le sprint **S1 — Spéc fonctionnelles**.

## Documents

- `analyse-note-lancement.md` : extraction des besoins explicites de HomeSkolar ;
- `perimetre-fonctionnel.md` : fonctionnalités incluses et éléments exclus ;
- `specifications-fonctionnelles.md` : version validée de la section fonctionnelle ;
- `controle-validation.md` : vérification de la couverture des besoins ;
- `exports/` : fichiers de travail exportés, dont le cahier des charges ODT intermédiaire.

## Règles appliquées

- la note de lancement HomeSkolar est la seule source fonctionnelle utilisée ;
- chaque fonctionnalité doit répondre à un besoin explicite ;
- aucune hypothèse ni fonctionnalité supplémentaire n'est ajoutée ;
- la rédaction reste compréhensible par un client non technique ;
- le cahier des charges complet doit rester limité à 2 ou 3 pages.
