# Backlog produit

Le [Backlog Produit initial](backlog-produit.md) contient les User Stories, leurs critères d’acceptation, leur priorité, leur estimation et leurs dépendances explicites.

Le [Kanban du Backlog Produit](kanban.md) présente leur ordre de réalisation et leur état d’avancement dans les colonnes « À faire », « En cours » et « Terminé ».

## Convention d’estimation

Les estimations sont exprimées en **jours idéaux**. Elles incluent développement, tests, revue et documentation. Elles servent à comparer l’effort relatif ; la planification calendaire dépendra de la capacité de l’équipe.

## Priorités

- **P0 — indispensable** : nécessaire à la première version utilisable ;
- **P1 — important** : forte valeur, planifiée après le socle P0 ;
- **P2 — amélioration** : utile mais différable sans bloquer le produit.
