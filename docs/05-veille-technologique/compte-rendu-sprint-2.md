# Compte rendu du Sprint S2 — Veille technologique

## Périmètre traité

Tickets OCPY3-21 à OCPY3-31, dans leur ordre de création.

## Résultat

La veille a permis de traduire les fonctionnalités HomeSkolar en besoins techniques, de comparer les solutions front-end, back-end Python et de stockage selon une grille homogène, puis de retenir une architecture simple : Django 5.2 LTS, gabarits Django avec Bootstrap 5.3 et PostgreSQL.

Les alternatives React, Vue, Flask, FastAPI, SQLite et MySQL ont été étudiées. Elles restent techniquement valables, mais n’apportent pas d’avantage suffisant pour justifier une architecture plus complexe ou moins intégrée dans le périmètre actuel.

## Contrôles

- cohérence avec les spécifications fonctionnelles du Sprint S1 ;
- sources officielles et récentes ;
- au moins trois sources pertinentes ;
- avantages et inconvénients équilibrés ;
- choix compréhensible et défendable ;
- orientation laissée volontairement à un niveau supérieur, les détails étant réservés au Sprint S3.

## Suite

Le Sprint S3 pourra convertir cette orientation en spécifications techniques : architecture, composants, échanges, sécurité, modèle de données, stratégie de tests et déploiement.
